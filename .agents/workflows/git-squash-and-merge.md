# دليل دمج التعديلات بطريقة Squash and Merge

للحفاظ على سجل نظيف في فرع `main` يحتوي على الـ Commits الأساسية والمهمة فقط، وتجنب ظهور الـ Commits الصغيرة (مثل Checkpoints)، يجب اتباع هذه الطريقة المعتمدة لدمج التعديلات (Merge).

لديك طريقتان للقيام بذلك: من خلال سطر الأوامر (الـ Terminal) أو من خلال موقع GitHub.

## الطريقة الأولى: من خلال Terminal (الموصى بها أثناء التطوير المحلي)

1. **انتقل إلى الفرع الأساسي** الذي تريد دمج التعديلات فيه (مثلاً `dev` أو `main`):

    ```bash
    git checkout dev
    ```

2. **نفّذ أمر الدمج مع تحديد الخيار `--squash`**:
   استبدل `feature/your-branch-name` باسم الفرع الذي كنت تعمل عليه:

    ```bash
    git merge --squash feature/your-branch-name
    ```

    _(هذا الأمر سيقوم بجمع جميع التعديلات في الـ Staging Area ولن يقوم بعمل Commit بشكل تلقائي)_.

3. **قم بعمل Commit جديد شامل يصف كل التعديلات المضافة**:

    ```bash
    git commit -m "feat: [وصف مختصر للميزة الجديدة مثل: إضافة نظام إدارة المشتركين]"
    ```

4. **ارفع التعديلات إلى المستودع البعيد (GitHub)**:
    ```bash
    git push origin dev
    ```

## الطريقة الثانية: من خلال موقع GitHub (Pull Request)

إذا كنت تفضل استخدام واجهة GitHub، يمكنك اتباع الخطوات التالية:

1. قم برفع الفرع الخاص بك إلى GitHub:

    ```bash
    git push origin feature/your-branch-name
    ```

2. اذهب إلى مستودع المشروع على موقع GitHub.

3. اضغط على زر **Compare & pull request**، واكتب عنواناً ووصفاً يوضحان ما تم إنجازه.

4. اطلب المراجعة (إذا كان هناك مراجعون) أو أكمل الـ Pull Request.

5. **الخطوة الأهم**: عند الموافقة على الـ Pull Request والوصول لخطوة الـ Merge، **لا تضغط على الزر الأخضر (Merge pull request) مباشرةً**.

6. اضغط على السهم بجانب الزر الأخضر، واختر القائمة المنسدلة، ثم حدد الخيار **Squash and merge**.

7. ستظهر لك نافذة تطلب منك تعديل رسالة الكوميت، اكتب فيها وصف الميزة المضافة (سيتم تجاهل الـ Commits الصغيرة وجمعها في هذا الـ Commit).

8. أخيرًا، اضغط على **Confirm squash and merge**.
